## 1. 目的
本書は、CareerLaunch の candidate data areas について、
CX22073JW と app-side operational domain の
配置境界を固定するための policy である。

## 2. Policy
### 2-1. CX22073JW に置くもの
- reusable reference
- reusable template
- reusable checklist
- reusable policy/rubric
- reusable option master

### 2-2. app-side に置くもの
- user-specific document
- application-specific context
- interview-specific context
- revision history
- share state
- persona/background selected state

## 3. Why
理由は次の通り。

- CX22073JW は fixed/reference/template foundation だから
- CareerLaunch 実行データは runtime / user-specific / case-specific だから
- 両者を混ぜると基盤と運用データの境界が壊れるから

## 4. Audit Principle
ただし revision history / share state / interview context は
後追い検証に重要であるため、
app-side であっても削らず明示的に保持対象とする。

## 5. UI Continuity Principle
persona/background setting は
単なる見た目設定ではなく、
CareerLaunch 体験の継続性に効くため、
候補領域として正式に扱う。
