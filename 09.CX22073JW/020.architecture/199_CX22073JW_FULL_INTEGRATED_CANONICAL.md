- policy-isolated

## 12. Actual vs Canonical Migration Strategy
The migration strategy is:
- not full rebuild
- additive migration
- keep / add / rename-or-wrap / deprecate based control

### 12-1. Keep
Retain stable canonical-compatible actual assets.

### 12-2. Add
Add missing canonical assets as new objects.

### 12-3. Rename-or-Wrap
Use wrapper/view or naming adjustment where actual meaning is close.

### 12-4. Deprecate
Non-canonical or overlapping assets should be deprecated, not immediately dropped.

### 12-5. Replace
Only for locally dangerous objects with deep PK/FK or role conflicts.
