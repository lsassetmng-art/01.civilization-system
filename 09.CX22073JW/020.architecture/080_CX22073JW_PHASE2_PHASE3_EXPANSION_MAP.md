## 1. Purpose

## 2. Phase2 Expansion Focus
Phase2 should deepen areas that are naturally adjacent to phase1 health/nutrition foundations.

### 2-1. Exercise Reference Deepening
Targets:
- richer exercise master
- intensity reference
- goal tags
- body-part mapping refinement
- caution rules by condition/profile

Why next:
- direct adjacency to BodyMetrics and TrainingCoach
- same compare/recommend pattern as phase1

### 2-2. Lifestyle Benchmark Deepening
Targets:
- lifestyle reference profile
- sleep / steps / hydration benchmark tables
- habit score bands
- habit comment templates

Why next:
- extends current health compare model
- strong reuse across BodyMetrics / TrainingCoach / LifePlanner

### 2-3. Helper Logic Deepening
Targets:
- exercise compare helper
- lifestyle compare helper
- habit score helper
- deficit/improvement candidate helper

Why next:
- foundation pattern already proven in phase1

## 3. Phase3 Expansion Focus
Phase3 should broaden into life/legal/preparation domains.

### 3-1. Legal Support Deepening
Targets:
- legal topic hierarchy
- consultation checklist refinement
- document requirement refinement

### 3-2. Inheritance / End-of-Life Deepening
Targets:
- inheritance categories
- end-of-life preference categories
- preparation checklists
- shared preparation template assets

### 3-3. Life Event Deepening
Targets:
- life event template layers
- phase/stage/checklist structure
- cost/document linkage

## 4. Why Phase2 Before Phase3
Reason:
- phase2 reuses the existing health/nutrition logic pattern directly
- phase3 introduces broader new domain semantics
- stabilizing adjacent logic first reduces expansion drift

## 5. CareerLaunch Position
CareerLaunch reusable template/reference expansion belongs after phase3 broadening or in parallel as a separated reusable layer.
Its runtime workspace remains outside CX22073JW.

1. phase1 stabilization

3. phase3 legal/inheritance/end-of-life/life-event broadening
