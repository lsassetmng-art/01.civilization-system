# FOUNDATION SCOPE RULES

## 1. purpose
This document fixes what belongs in Foundation.

## 2. in-scope
Foundation may contain:
- top-level design principles
- worldview assumptions
- root-wide interpretation priorities
- non-domain-specific architectural rules
- high-level ownership and authority discipline
- irreversible root-level structure constraints

## 3. out-of-scope
Foundation must not contain:
- app screen rules
- domain runtime records
- domain-internal low-level workflows
- destination-specific portal route details
- product-specific feature packaging

## 4. writing rule
A Foundation statement should still make sense
even before naming a specific OS/app destination.

If a statement only makes sense inside one domain,
it probably belongs below Foundation.
