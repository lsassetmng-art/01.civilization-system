# FOUNDATION ESCALATION RULES

## 1. purpose
This document fixes when lower-layer issues should move upward to Foundation.

## 2. escalate to Foundation when
- the issue changes whole-system premise
- the issue affects multiple OS layers at the same top-level meaning
- the issue changes root-wide interpretation priorities
- the issue changes irreversible top-level structure

## 3. do not escalate to Foundation when
- the issue is only domain-specific
- the issue is only app-specific
- the issue is only portal-route-specific
- the issue is implementation-specific
- the issue can be resolved by the domain owner without changing top-level premise

## 4. anti-pattern
Do not send unresolved detail upward
just because it is difficult at the lower layer.

## 5. decision sentence
Only elevate what is truly root-wide and premise-level.
