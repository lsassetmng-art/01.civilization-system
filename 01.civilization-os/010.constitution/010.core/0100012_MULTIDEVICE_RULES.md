# ============================================================
# 0. PURPOSE
# ============================================================

本設計は、
CivilizationOS を単一端末専用ではなく、
以下の主要利用端末に対応する前提で定義する。

- iPhone
- Android
- PC
- タブレット

本ルールの目的は、
端末差により意味、権限、状態遷移、監査、同期結果が崩れないことを
最上位ルールとして明文化することである。

# ============================================================
# 1. CONCLUSION
# ============================================================

結論:
- CivilizationOS は最初からマルチデバイス対応前提で設計する
- iPhone / Android / PC / タブレットを正式対象とする
- 端末別UI最適化は許容する
- ただし意味、権限、状態、監査、同期、公開境界は共通正本に従う
- 端末差は presentation 差に留め、domain truth を分岐させない

# ============================================================
# 2. TARGET DEVICES
# ============================================================

正式対応対象:
- iPhone
- Android
- PC
- タブレット

補足:
- iPhone / Android はモバイル端末として扱う
- PC は高密度作業端末として扱う
- タブレットは中間利用端末として扱う

# ============================================================
# 3. CORE RULES
# ============================================================

## 3-1. Same Meaning, Different Presentation
端末ごとに画面構成、配置、操作導線が異なってもよい。
ただし、以下は共通でなければならない。

- 用語意味
- 状態意味
- 権限意味
- 承認意味
- 監査意味
- 同期意味
- 公開意味

## 3-2. Shared Domain Truth
正本データは端末別に分岐してはならない。
同一アカウントはどの端末から利用しても、
同じ正本状態を参照しなければならない。

## 3-3. Device-Optimized UI
UI は端末ごとに最適化してよい。
ただしUI差により、
意味や権限や state machine が変化してはならない。

## 3-4. Fail Closed Across Devices
端末差により validation や権限制御が抜けてはならない。
どの端末でも fail closed を維持する。

# ============================================================
# 4. DEVICE RULES
# ============================================================

## 4-1. iPhone / Android
- モバイル利用前提で最適化する
- 通知起点導線を重視する
- 縦長画面前提で構成する
- 不安定回線やオフラインを考慮する
- 軽入力、確認、承認、即応を重視する

## 4-2. PC
- 一覧性、管理作業性、高密度表示を重視する
- 長文入力や比較作業を重視する
- 一括操作や複数情報の同時確認を重視する

## 4-3. タブレット
- 閲覧性と軽編集の両立を重視する
- 会議、現場、接客などの中間利用を想定する
- 縦横両利用を考慮する

# ============================================================
# 5. COMMON GUARANTEES
# ============================================================

すべての端末で保証すべき事項:
- 同一権限判定
- 同一 validation
- 同一 review result
- 同一 audit trail
- 同一 sync result
- 同一 publish boundary
- 同一 security policy

# ============================================================
# 6. CROSS-DEVICE CONTINUITY
# ============================================================

同一ユーザーが端末を跨いで継続利用できるよう、
以下を維持する。

- draft continuity
- notification continuity
- pending continuity
- unread continuity
- attachment continuity
- recent activity continuity

# ============================================================
# 7. SECURITY RULE
# ============================================================

端末差があっても、
以下の security rule は共通とする。

- authentication
- authorization
- session handling
- masking
- audit logging
- denied handling
- review-required handling

端末別に権限を緩めることは禁止する。

# ============================================================
# 8. FINAL STATEMENT
# ============================================================

CivilizationOS は、
iPhone、Android、PC、タブレット対応を
最初から想定したマルチデバイス前提の設計とする。

端末ごとのUI差は許容するが、
意味、状態、権限、監査、同期、公開境界は
常に共通正本に従わなければならない。
