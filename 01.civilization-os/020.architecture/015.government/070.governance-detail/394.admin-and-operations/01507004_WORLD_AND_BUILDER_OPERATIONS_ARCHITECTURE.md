# ============================================================
# 1. PURPOSE
# ============================================================

Define the canonical architecture
for world correction,
map supervision,
builder supervision,
and world-structure operations
inside Civilization.


# ============================================================
# 2. CORE RESPONSIBILITIES
# ============================================================

This tool family must support:

- state-builder supervision
- city-builder supervision
- world-placement review
- state / city correction posture
- district / facility structural review
- builder-result hold or approval where required
- map truth correction where canonical policy allows


# ============================================================
# 3. ORDINARY BUILDER SEPARATION
# ============================================================

World operations tools
must remain separate from:

- ordinary state builder UI
- ordinary city builder UI
- ordinary map browsing UI

These tools supervise,
correct,
or override
builder outcomes when authorized.


# ============================================================
# 4. FINAL RULE
# ============================================================

World and builder operations architecture must remain:

- map-truth-aware
- builder-supervisory
- correction-capable
- authority-separated
- traceable
