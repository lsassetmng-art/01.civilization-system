# ============================================================
# 1. PURPOSE
# ============================================================

Defines the canonical architecture
for the Civilization digital-only marketplace.

This architecture defines:

- civilization-wide marketplace core
- nation-specific marketplace variation
- company-specific store variation
- digital product categories
- listing and moderation structure
- order, settlement, and access structure
- trust, review, and report structure
- marketplace UI entry structure


# ============================================================
# 2. CORE PRINCIPLE
# ============================================================

Digital Marketplace must separate:

- global marketplace rules
- nation restriction and tax rules
- seller and company store rules
- product and listing rules
- moderation and illegality rules
- UI and navigation rules

No single layer may silently redefine
another layer's authority.


# ============================================================
# 3. THREE-LAYER MODEL
# ============================================================

The marketplace consists of:

- Civilization Global Marketplace Layer
- Nation Marketplace Policy Layer
- Seller and Company Store Layer

Civilization Global Marketplace Layer:
- common category structure
- common listing logic
- common trust and report logic
- common purchase and access logic

Nation Marketplace Policy Layer:
- sale restriction
- view restriction
- tax and fee differences
- religion and culture restriction
- visibility and ranking differences

Seller and Company Store Layer:
- company store identity
- seller permissions
- store presentation
- seller-specific product templates
- direct store routing


# ============================================================
# 4. DIGITAL-ONLY RULE
# ============================================================

Marketplace scope in this architecture
is digital-only.

No physical shipment logic is required
for the canonical base layer.

Marketplace objects may include:
- UI themes
- event decoration packs
- digital books
- digital art
- music and voice assets
- text templates
- story packs
- company digital products
- nation theme packs
- ceremony assets
- service access rights


# ============================================================
# 5. FINAL RULE
# ============================================================

The digital marketplace must remain
globally coherent,
nation-aware,
seller-aware,
and implementation-ready.
