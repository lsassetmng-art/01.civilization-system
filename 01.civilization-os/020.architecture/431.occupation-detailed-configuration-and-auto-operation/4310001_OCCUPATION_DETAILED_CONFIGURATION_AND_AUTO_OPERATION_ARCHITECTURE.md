# ============================================================
# 1. PURPOSE
# ============================================================

Defines the canonical common architecture
for occupation-oriented detailed UI,
configuration-driven auto operation,
KPI observation,
alert handling,
and intervention loops inside Civilization.

This architecture is future-oriented.

It is not limited only to currently active systems.


# ============================================================
# 2. TARGET OCCUPATIONS
# ============================================================

This architecture may be applied to:

- retail
- food_service
- agriculture
- livestock
- fishery
- food_processing
- manufacturing
- railway_operation
- bus_operation
- logistics_operation
- hospital_operation
- school_operation
- construction_operation
- real_estate_management
- media_operation
- military_base_operation
- administration_operation


# ============================================================
# 3. CORE PRINCIPLE
# ============================================================

Occupation systems must not require
constant repetitive micro-control.

The canonical loop is:

detailed configuration
-> auto operation
-> KPI observation
-> alert detection
-> intervention
-> auto operation resume

This architecture must remain compatible
with Earth-like real-time progression.


# ============================================================
# 4. COMMON STRUCTURE
# ============================================================

Every occupation detail system
must support at least:

- overview surface
- detailed configuration surface
- auto operation rule binding
- KPI observation
- alert visibility
- intervention execution
- inspection or audit event linkage
- nation and region difference awareness


# ============================================================
# 5. DIFFERENCE PRINCIPLE
# ============================================================

Each occupation may expose different
configuration categories and KPI sets.

However, the common control loop
must remain structurally aligned.


# ============================================================
# 6. AUTO OPERATION PRINCIPLE
# ============================================================

Auto operation is the normal default.

Detailed control exists to:

- improve efficiency
- improve quality
- improve safety
- improve resilience
- improve social contribution
- improve local fit
- improve profitability
- improve compliance


# ============================================================
# 7. INTERVENTION PRINCIPLE
# ============================================================

Intervention must be explicit,
bounded,
and resumable.

Intervention must not replace
the normal operation loop with
constant manual burden.


# ============================================================
# 8. INSPECTION AND AUDIT PRINCIPLE
# ============================================================

Occupation systems may become subjects of:

- inspection
- audit
- review
- compliance checks
- safety checks
- quality checks

Inspection or audit design
must be occupation-aware,
but structurally align with the common framework.


# ============================================================
# 9. NATION AND REGION PRINCIPLE
# ============================================================

Occupation systems must be affected by:

- nation policy
- region profile
- district profile
- supply environment
- labor environment
- infrastructure condition
- emergency conditions

Auto operation must read these conditions.


# ============================================================
# 10. FINAL RULE
# ============================================================

Civilization occupation management
must be configuration-driven,
auto-operated,
KPI-observed,
and intervention-based,
with a reusable common structure
for future expansion.
