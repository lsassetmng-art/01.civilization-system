# ============================================================
# 1. PURPOSE
# ============================================================

Define the canonical architecture
for autonomous movement
and autonomous activity
inside Civilization.


# ============================================================
# 2. CORE PRINCIPLE
# ============================================================

Autonomous behavior
is the architecture family
that allows actors to move and act
without direct manual input
at every step.

This includes:

- movement
- attendance
- facility usage
- work activity
- consumption activity
- event participation
- rest and return activity


# ============================================================
# 3. PRIMARY ACTORS
# ============================================================

Autonomous behavior must support:

- user under assisted automation posture where applicable
- Ai(Human) as persistent autonomous actor

These two families may differ
in control priority,
but share behavior architecture.


# ============================================================
# 4. BEHAVIOR DRIVERS
# ============================================================

Behavior may be driven by:

- current schedule
- role obligations
- need posture
- event invitation or eligibility
- location and route availability
- work assignment
- school assignment
- residence posture
- emergency or risk posture

Behavior is not random movement only.


# ============================================================
# 5. MOVEMENT RELATION
# ============================================================

Autonomous behavior must support movement between:

- residence
- workplace
- school
- facility
- event venue
- market location
- safe location where emergency posture requires

Movement and activity remain connected,
but are not identical.


# ============================================================
# 6. FINAL RULE
# ============================================================

Autonomous behavior architecture must remain:

- actor-aware
- schedule-aware
- location-aware
- role-aware
- need-aware
- emergency-compatible
