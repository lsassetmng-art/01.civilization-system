# ============================================================
# 1. PURPOSE
# ============================================================

Define the canonical runtime architecture
for reflecting world-state change
onto map and world-visible presentation
inside Civilization.


# ============================================================
# 2. CORE PRINCIPLE
# ============================================================

World change must be reflectable
onto map and visual world layers.

Representative reflected changes include:

- facility state change
- infrastructure outage or restoration
- city growth or decline
- congestion and traffic posture
- disaster effect
- actor presence density where applicable


# ============================================================
# 3. FINAL RULE
# ============================================================

Map reflection and world visual runtime architecture must remain:

- state-reflective
- map-aware
- world-visible
- compatible with simulation state
