# ============================================================
# 1. PURPOSE
# ============================================================

Define the canonical architecture
for facility-level autonomous progression
inside Civilization.


# ============================================================
# 2. CORE PRINCIPLE
# ============================================================

Facilities must be treated as
autonomously progressing operational entities.

Facility progression may include:

- operating
- underused
- overloaded
- degraded
- damaged
- under maintenance
- restored
- abandoned where applicable
- repurposed where applicable


# ============================================================
# 3. INPUTS
# ============================================================

Representative facility inputs include:

- actor usage
- staffing
- supply and inventory
- infrastructure dependency
- maintenance burden
- environmental burden
- disaster burden
- economic demand
- policy restriction


# ============================================================
# 4. OUTPUTS
# ============================================================

Representative facility outputs include:

- service availability
- failure posture
- quality decline
- queue burden
- maintenance requirement
- closure risk
- linked-facility impact


# ============================================================
# 5. FINAL RULE
# ============================================================

Facility autonomous operation architecture must remain:

- facility-scale
- service-aware
- dependency-aware
- maintenance-aware
- failure-capable
