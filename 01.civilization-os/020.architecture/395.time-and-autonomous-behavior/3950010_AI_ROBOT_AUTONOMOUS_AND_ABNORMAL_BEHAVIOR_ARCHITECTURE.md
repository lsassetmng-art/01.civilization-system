# ============================================================
# 1. PURPOSE
# ============================================================

Define the canonical architecture
for normal autonomous operation
and abnormal behavior
of Ai(Robot) inside Civilization.


# ============================================================
# 2. CORE PRINCIPLE
# ============================================================

Ai(Robot) behavior
must remain separate from Ai(Human) behavior.

Ai(Robot) abnormal behavior
may originate from:

- malware or virus contamination
- logic corruption
- experimental logic change
- faulty update
- control-system failure
- communication failure
- authentication failure


# ============================================================
# 3. NORMAL ROBOT BEHAVIOR
# ============================================================

Representative normal robot behavior includes:

- assigned task execution
- route movement
- facility service support
- logistics support
- public-service support where authorized
- industrial or domestic support behavior


# ============================================================
# 4. ABNORMAL ROBOT BEHAVIOR
# ============================================================

Representative abnormal robot behavior includes:

- command noncompliance
- excessive execution
- wrong-target execution
- repeated loop behavior
- unsafe execution
- destructive or harmful operation
- coordinated malfunction
- propagation behavior across linked units


# ============================================================
# 5. CONTAINMENT AND RECOVERY
# ============================================================

Civilization must support posture for:

- isolation
- shutdown
- rollback
- quarantine
- recovery
- revalidation of restored units

Abnormal robot behavior
must not remain unmanaged.


# ============================================================
# 6. FINAL RULE
# ============================================================

Ai(Robot) autonomous and abnormal behavior architecture must remain:

- robot-distinct
- contamination-aware
- override-aware
- containment-capable
- recovery-capable
