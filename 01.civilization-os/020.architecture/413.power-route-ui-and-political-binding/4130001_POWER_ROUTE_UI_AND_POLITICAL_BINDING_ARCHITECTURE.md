# ============================================================
# 1. PURPOSE
# ============================================================

Defines the canonical binding architecture
between:

- political power and election play
- grand play UI integration
- nation type and power acquisition design

This architecture defines:

- how governance_type affects political play entry
- how nation-specific override affects visible routes
- how legitimacy appears in political UI
- how power ascent routes become actionable
- how blocked and crisis-opened routes are surfaced


# ============================================================
# 2. CORE PRINCIPLE
# ============================================================

Power acquisition must not remain
a hidden background rule.

It must appear through:

- political entry surfaces
- succession or election surfaces
- personal rise-and-fall surfaces
- grand play summary surfaces

The user must be able to understand:

- what route is available
- why it is available
- why it is blocked
- what legitimacy source supports it
- what crisis state may open additional routes


# ============================================================
# 3. FINAL RULE
# ============================================================

Political play and grand play UI
must expose power acquisition truth
without collapsing nation differences.
