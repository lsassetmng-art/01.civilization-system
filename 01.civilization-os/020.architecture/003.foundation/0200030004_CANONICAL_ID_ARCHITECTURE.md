# CANONICAL ID ARCHITECTURE

status: canonical
layer: architecture
domain: foundation

## PURPOSE
Define canonical identifier rules.

## RULES
- Every major entity must have a stable canonical ID
- IDs must be immutable after creation
- Human-readable names are not identifiers
- Cross-domain references must use canonical IDs only

## TARGET ENTITIES
- world
- nation
- city
- player
- persona
- company
- policy
- treaty
- runtime job
