# WORLD FOUNDATION BOUNDARY ARCHITECTURE

status: canonical
layer: architecture
domain: foundation

## PURPOSE
Define the boundary between universal world structure and domain-specific logic.

## RULES
- World existence model belongs to foundation
- Nation, company, law, war, persona remain domain-owned
- Cross-domain references must pass through explicit IDs
- No hidden cross-layer mutation is allowed

## BOUNDARY EXAMPLES
Foundation:
- world identity
- timeline base
- canonical namespace
- domain registry

Not foundation:
- taxation rules
- company behavior
- persona emotion logic
- battle resolution
