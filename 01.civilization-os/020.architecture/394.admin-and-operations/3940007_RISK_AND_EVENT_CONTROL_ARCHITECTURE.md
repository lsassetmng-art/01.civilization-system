# ============================================================
# 1. PURPOSE
# ============================================================

Define the canonical architecture
for supervision,
review,
and controlled intervention
over risk and event systems
inside Civilization.


# ============================================================
# 2. CORE RESPONSIBILITIES
# ============================================================

This tool family must support:

- war-risk visibility
- epidemic-risk visibility
- famine-risk visibility
- mortality-risk visibility
- event eligibility supervision
- event hold / release posture where policy allows
- emergency trigger or suppression posture where policy allows
- audit visibility of intervention actions


# ============================================================
# 3. SIMULATION SEPARATION
# ============================================================

Risk and event control tools
supervise simulation posture,
but must not silently redefine
core simulation truth
without explicit operator action and traceability.


# ============================================================
# 4. FINAL RULE
# ============================================================

Risk and event control architecture must remain:

- risk-aware
- event-aware
- intervention-capable
- authority-separated
- traceable
