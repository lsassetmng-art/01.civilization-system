# ============================================================
# 1. PURPOSE
# ============================================================

Define the canonical architecture
for system-only infrastructure supervision,
placement,
extension,
and correction tools
inside Civilization.


# ============================================================
# 2. CORE RESPONSIBILITIES
# ============================================================

This tool family must support:

- infrastructure placement
- infrastructure extension
- node / route / line adjustment
- supply-impact review
- outage-impact review
- dependency-impact review
- system-only infrastructure confirmation
- placement history visibility


# ============================================================
# 3. SYSTEM-ONLY PRINCIPLE
# ============================================================

Infrastructure placement and structural change
are system-only capabilities.

They must not be available
to ordinary users
or ordinary facility operators.


# ============================================================
# 4. FINAL RULE
# ============================================================

Infrastructure system operations architecture must remain:

- system-only
- placement-capable
- validation-based
- impact-aware
- traceable
