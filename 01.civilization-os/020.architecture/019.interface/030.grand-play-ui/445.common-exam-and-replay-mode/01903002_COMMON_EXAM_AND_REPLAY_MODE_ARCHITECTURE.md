# ============================================================
# 1. PURPOSE
# ============================================================

Defines the canonical architecture
for common exams,
official exam attempts,
auto-generated answers,
user-side answer correction,
automatic submission,
score-based result handling,
and unlimited replay mode after official completion.

This architecture covers:

- school exam
- individual exam
- common exam

with special focus on common exams
similar to national center-style exams.


# ============================================================
# 2. EXAM CLASSIFICATION
# ============================================================

Civilization exams are classified into:

- school exam
- individual exam
- common exam

School exam:
Used inside a school progression context.

Individual exam:
Used for a specific school,
qualification,
company,
or office route.

Common exam:
Used as a shared score-bearing exam
for multiple education or route targets.


# ============================================================
# 3. COMMON EXAM PRINCIPLE
# ============================================================

A common exam is not only pass/fail oriented.

A common exam primarily returns:

- official score
- rank or equivalent score profile
- usable route range
- downstream route eligibility

This exam may be used by:

- higher education admission
- scholarship evaluation
- public service foundation route
- other bound progression systems


# ============================================================
# 4. ANSWER PRINCIPLE
# ============================================================

Exam answers are auto-generated by default.

Auto-generated answers are derived from:

- actor ability
- learning status
- qualification preparation state
- school history
- health and concentration state
- permitted item effects

The user may review and edit the answers
before final submission.

If time expires,
the current answer state is automatically submitted.


# ============================================================
# 5. OFFICIAL ATTEMPT PRINCIPLE
# ============================================================

An official attempt is the only attempt
that may produce formal score
and formal progression result.

Official attempts may produce:

- pass/fail
- score
- route unlock
- route restriction
- progression trigger


# ============================================================
# 6. REPLAY MODE PRINCIPLE
# ============================================================

After an official attempt is completed,
replay mode may be unlocked.

Replay mode allows:

- unlimited retry
- repeated solving
- review and training
- answer experimentation

Replay mode must not change:

- official score
- official pass/fail
- formal public record
- formal qualification state
- formal route result


# ============================================================
# 7. FINAL RULE
# ============================================================

Civilization exam systems
must separate official result generation
from unlimited replay-based review,
while keeping answer handling convenient by default.
