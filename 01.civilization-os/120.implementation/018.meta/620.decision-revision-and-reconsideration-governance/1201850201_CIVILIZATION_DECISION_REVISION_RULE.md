## 1. Purpose

Defines the canonical rule for revising the current final decision pack.

## 2. Current Baseline

The current baseline decision posture is:

- keep phase1 open in design-only mode
- do not resume execution yet
- do not prepare transition yet
- preserve one narrowed phase2 candidate

## 3. Revision Rule

The baseline may be changed only by an explicit later decision update.

A valid revision must specify:

- which baseline field changes
- why it changes
- which prior decision it supersedes
- whether the change affects
  - phase1 close posture
  - execution resume posture
  - transition posture
  - phase2 entry posture

## 4. Hard Rule

No informal interpretation, summary, or side memo may override the final decision pack.

## 5. Acceptance Checklist

- baseline fixed
- revision rule fixed
- no-informal-override rule fixed
