## 1. Purpose

Defines the maintenance protocol for preserving the canonical surface.

## 2. Maintenance Objectives

Maintenance should preserve:

- one clear first-reference path
- one clear change path
- one clear current posture summary
- one clear successor path

## 3. Canonical Maintenance Actions

Allowed maintenance actions:

- navigation cleanup
- summary alignment
- status synchronization
- successor-link synchronization
- audit refresh

## 4. Forbidden Maintenance Actions

Do not do the following under maintenance:

- silently revise the decision posture
- silently change reopen criteria
- silently widen a phase2 candidate
- silently reinterpret freeze state

## 5. Maintenance Trigger Examples

Maintenance may be triggered by:

- new canonical pack added
- status link drift detected
- navigation drift detected
- audit mismatch detected

## 6. Hard Rule

Maintenance is not decision revision.
Decision revision must use the governed change path.

## 7. Acceptance Checklist

- maintenance objectives fixed
- allowed actions fixed
- forbidden actions fixed
- no-maintenance-as-revision rule fixed
