## 1. Purpose

Defines how future sessions must continue from the current canonical surface.

## 2. Continuity Objective

Future sessions must preserve:

- the current frozen posture
- the current top-level reading order
- the current governed change path
- the current no-change default

## 3. Continuity Start Rule

Any future session should begin by reading in this order:

1. current posture one-page
2. canonical posture handoff
3. canonical baseline snapshot
4. operator entrypoint master

## 4. Continuity Change Rule

If a future session intends change, it must continue by reading:

- decision revision rule
- the correct reconsideration or update template
- the minimum change control set

## 5. Hard Rule

Do not start a future session from isolated old execution or old design detail
without first anchoring to the current canonical surface.

## 6. Acceptance Checklist

- continuity objective fixed
- continuity start rule fixed
- continuity change rule fixed
- no-old-detail-first rule fixed
