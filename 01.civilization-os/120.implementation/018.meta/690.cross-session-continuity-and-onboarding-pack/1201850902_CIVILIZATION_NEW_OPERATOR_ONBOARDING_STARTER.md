## 1. Purpose

Provides the shortest safe onboarding path for a new operator.

## 2. Starter Questions

A new operator should answer first:

1. what is the current posture?
2. is change intended or not intended?
3. what is the preserved phase2 candidate?
4. what document is the first deep source if clarification is needed?

## 3. Starter Reading Order

Recommended starter reading order:

1. current posture one-page
2. canonical posture handoff
3. final decision pack integrated status
4. operator entrypoint master
5. steady-state and change-initiation pack integrated status

## 4. Starter Summary

Current starter summary:

- phase1 open
- design-only
- no execution resume
- no transition preparation
- one narrowed phase2 candidate preserved
- governed change only

## 5. Hard Rule

A new operator should not attempt reinterpretation
before the starter reading order is completed.

## 6. Acceptance Checklist

- starter questions fixed
- starter reading order fixed
- starter summary fixed
- no-pre-read reinterpretation rule fixed
