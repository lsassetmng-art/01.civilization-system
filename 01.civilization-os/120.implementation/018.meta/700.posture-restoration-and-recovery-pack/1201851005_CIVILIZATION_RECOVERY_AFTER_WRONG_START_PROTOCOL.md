## 1. Purpose

Defines the exact protocol when a future session starts from the wrong place.

## 2. Wrong Start Examples

Wrong start includes:

- starting from old partial execution records first
- starting from intermediate exact-design packs first
- starting from archive references first
- starting from one quick summary and stopping there

## 3. Recovery Protocol

When wrong start is detected:

1. stop deeper interpretation
2. return to current posture one-page
3. continue through the minimum restoration bundle
4. reclassify intent as:
   - read-only
   - maintenance / errata
   - governed semantic change
5. proceed only after reclassification is explicit

## 4. Recovery Output Fields

Required fields:

- recovery_protocol_id
- wrong_start_type
- restoration_bundle_used
- recovered_posture_summary
- reclassified_intent
- recovery_summary_text

## 5. Hard Rule

Wrong start recovery is mandatory before any new conclusion is adopted.
