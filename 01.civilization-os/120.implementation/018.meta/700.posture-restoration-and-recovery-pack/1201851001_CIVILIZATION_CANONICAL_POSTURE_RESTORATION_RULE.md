## 1. Purpose

Defines how to restore the current canonical posture
after drift, confusion, or wrong-entry handling.

## 2. Current Canonical Posture To Restore

Restore the following posture:

- phase1 remains open
- phase1 remains design-only
- execution does not resume
- transition preparation does not start
- one narrowed phase2 candidate is preserved
- any future semantic change must use governed paths

## 3. Restoration Rule

When drift or uncertainty is detected, restore posture by re-reading in this order:

1. current posture one-page
2. canonical posture handoff
3. canonical baseline snapshot
4. final decision pack integrated status
5. operator entrypoint master

## 4. Restoration Completion Rule

Restoration is complete only when:

- the current posture matches the frozen baseline
- no accidental semantic reinterpretation remains
- the next intended action is reclassified through the governed path

## 5. Hard Rule

Do not attempt restoration by editing summary documents first.
Restore by re-anchoring to the canonical surface first.
