## 1. Purpose

Defines the canonical contract for using the late-stage CivilizationOS surface.

## 2. Canonical Usage Objective

The usage objective is to ensure that future handling always begins from:

- the active canonical surface
- the current frozen baseline
- the correct governed change path when change is intended

## 3. Canonical Usage Contract

Every future operator or session should:

1. start from the current posture one-page or canonical handoff
2. confirm whether intent is read-only or change-intended
3. stay in steady-state mode when no valid trigger exists
4. use governed change initiation only when a valid trigger exists
5. restore posture first if drift or wrong-start handling is detected

## 4. Required Usage Outputs

Required outputs when usage becomes active:

- usage_context_id
- current_intent_class
- baseline_confirmed_flag
- valid_trigger_flag
- selected_operating_path
- usage_summary_text

## 5. Hard Rule

The canonical surface is not optional orientation.
It is the required usage contract.

## 6. Acceptance Checklist

- canonical usage objective fixed
- canonical usage contract fixed
- required usage outputs fixed
- no-optional-orientation rule fixed
