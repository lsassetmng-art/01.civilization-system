## 1. Purpose

Defines the safe refusal posture when future action is not yet justified.

## 2. Safe Default Refusal Cases

Refuse to deepen action when:

- canonical surface was not read first
- baseline is not confirmed
- prohibited shortcut is active
- semantic vs non-semantic classification is unresolved
- no valid trigger exists for change
- recovery should happen first

## 3. Refusal Result Rule

In refusal cases, the correct result is:

- do not proceed
- re-anchor to canonical surface
- restore posture if needed
- use governed path only after prerequisites are satisfied

## 4. Hard Rule

Safe refusal is not blockage.
It is the correct preservation behavior for canonical stability.

## 5. Acceptance Checklist

- safe refusal cases fixed
- refusal result rule fixed
- preservation-not-blockage rule fixed
