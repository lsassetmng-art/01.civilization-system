## 1. Purpose

Defines the shortcut patterns that are prohibited for future handling.

## 2. Prohibited Shortcut Families

shortcut_01:
- name: old_detail_first
- description: starting from older detailed packs instead of the active canonical surface

shortcut_02:
- name: quick_summary_override
- description: treating a compact summary as if it overrides canonical decision packs

shortcut_03:
- name: candidate_equals_approval
- description: treating preserved phase2 candidate as approved phase2 entry

shortcut_04:
- name: drift_as_new_truth
- description: repeating a drifted interpretation until it sounds canonical

shortcut_05:
- name: maintenance_as_revision
- description: using maintenance or errata to change semantic posture

shortcut_06:
- name: recovery_as_change
- description: converting restoration flow directly into new posture change without governance

shortcut_07:
- name: verbal_intent_only
- description: starting semantic change from talk alone without checklist, template, or gate

## 3. Shortcut Handling Rule

When a shortcut family is detected, the correct response is:

- stop deeper action
- re-anchor to canonical surface
- reclassify intent
- use governed path if change is truly intended

## 4. Hard Rule

Shortcut convenience is never a valid substitute
for canonical entry, restoration, or governed change.
