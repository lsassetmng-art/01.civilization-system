## 1. Purpose

Defines the single best entrypoint order for future operators.

## 2. Entry Order

Recommended operator entry order:

1. read current posture one-page
2. read canonical posture handoff
3. read canonical baseline snapshot
4. decide whether intent is:
   - read only
   - governed change
5. if read only, stop
6. if governed change, move into governance and templates

## 3. Escalation Rule

Escalate into deeper packs only when the operator needs:

- formal decision detail
- governed reconsideration
- template-based change request
- audit/maintenance reasoning

## 4. Hard Rule

Do not begin from old intermediate design packs
unless traceability investigation is explicitly needed.

## 5. Acceptance Checklist

- entry order fixed
- escalation rule fixed
- no-old-pack-first rule fixed
