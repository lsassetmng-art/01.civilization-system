## 1. Purpose

Defines the rule that freezes the current canonical baseline.

## 2. Current Frozen Baseline

The current frozen baseline is:

- phase1 remains open
- phase1 remains design-only
- execution does not resume
- transition preparation does not start
- one narrowed phase2 candidate is preserved

## 3. Freeze Rule

Treat the current posture as frozen baseline until an explicit later
decision update supersedes it.

## 4. Allowed Change Path

Allowed change path only:

1. use reconsideration governance
2. use the correct request template
3. emit an explicit decision update
4. ledger the update

## 5. Hard Rule

No ad-hoc note, chat summary, or inferred intent may thaw the baseline.

## 6. Acceptance Checklist

- frozen baseline fixed
- allowed change path fixed
- no-ad-hoc-thaw rule fixed
