## 1. Purpose

Provides the checklist required before any governed change starts.

## 2. Initiation Checklist

Before starting governed change, confirm:

1. current posture one-page read
2. canonical posture handoff read
3. baseline snapshot confirmed
4. valid trigger identified
5. correct scenario family selected
6. correct template selected
7. semantic vs non-semantic path distinguished
8. no silent widening implied

## 3. Required Checklist Outputs

Required checklist outputs:

- initiation_check_id
- trigger_name
- selected_change_path
- selected_template_ref
- semantic_change_flag
- widening_risk_note
- checklist_summary_text

## 4. Hard Rule

Do not open a change path with only verbal intent and no checklist trace.

## 5. Acceptance Checklist

- initiation checklist fixed
- required outputs fixed
- no-verbal-only-initiation rule fixed
