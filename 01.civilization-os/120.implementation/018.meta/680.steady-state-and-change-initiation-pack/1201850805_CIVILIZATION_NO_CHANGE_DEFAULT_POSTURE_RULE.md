## 1. Purpose

Defines the default rule that applies when no valid change trigger exists.

## 2. Default No-Change Posture

If no valid trigger exists, the default posture is:

- no new decision update
- no reopen
- no execution resume reconsideration
- no phase2 entry reconsideration
- no new semantic interpretation shift

## 3. Default Action Rule

When uncertain, default to:

- read-only handling
- frozen baseline preservation
- canonical surface preservation

## 4. Hard Rule

Uncertainty is resolved in favor of stability, not movement.

## 5. Acceptance Checklist

- default no-change posture fixed
- default action rule fixed
- stability-first rule fixed
