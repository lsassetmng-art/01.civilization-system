# ============================================================
# 1. CORE POLICY
# ============================================================

Company list, analytics, evaluation, and listing-readiness
must remain explicit and auditable.


# ============================================================
# 2. CREATION EVALUATION POLICY
# ============================================================

A company should become evaluable at creation time.

Initial evaluation is mandatory as a system concept.


# ============================================================
# 3. LISTING POLICY
# ============================================================

Listing or going public is not immediate.

It should only be available after threshold criteria are satisfied.
Listed status must remain explicit and separate from ordinary company status.


# ============================================================
# 4. FINAL POLICY RULE
# ============================================================

Company UI policy in CivilizationOS must be:
- list-aware
- analytics-aware
- evaluation-aware
- threshold-aware
- auditable
