purpose:
Define the flow for retail digital products
inside CivilizationOS.


# ============================================================
# 1. PRODUCT SALE FLOW
# ============================================================

product listing
-> moderation / publication validation
-> purchase
-> entitlement creation
-> license rule attachment
-> product activation or availability


# ============================================================
# 2. APP ACTIVATE FLOW
# ============================================================

app product acquired
-> app install or activate
-> origin OS / integration mode interpreted
-> delegated runtime/access policy target identified
-> app becomes usable under delegated OS policy where applicable


# ============================================================
# 3. LICENSE / ENTITLEMENT CHANGE FLOW
# ============================================================

license or entitlement event
-> compatibility and transferability validation
-> entitlement update
-> app/runtime availability refresh


# ============================================================
# 4. FINAL FLOW RULE
# ============================================================

Retail digital product flow must remain:
- entitlement-aware
- license-aware
- origin-aware
- delegation-aware
