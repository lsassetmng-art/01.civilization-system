# ============================================================
# 1. PURPOSE
# ============================================================

This additive note clarifies
cross-device platform support,
multilingual support,
and multi-currency support
for StreamingOS.

This note is additive only.
It does not replace existing StreamingOS definitions.

# ============================================================
# 2. CROSS-DEVICE PLATFORM SUPPORT
# ============================================================

StreamingOS shall be designed
as a cross-device platform.

Supported device classes:
- iPhone
- Android
- PC
- tablet

Interpretation:
- iPhone and Android are first-class viewer-side and creator-side client targets
- PC is a first-class target for creator operation,
  moderation,
  analytics,
  review,
  and corporate oversight
- tablet is a first-class target for creator operation,
  moderation,
  oversight,
  and viewer-side rich interaction

Canonical statement:
StreamingOS is not limited to a single device family.
It shall be usable across smartphone,
PC,
and tablet environments.

# ============================================================
# 3. MULTILINGUAL SUPPORT
# ============================================================

StreamingOS shall support multilingual operation.

Multilingual support applies to:
- UI labels
- settings labels
- notification text
- moderation / review reason labels
- creator-facing dashboard labels
- viewer-facing discovery labels
- category / tag / platform-visible text where applicable

Design principle:
Multilingual support shall be treated
as a common capability,
not as ad hoc per-screen wording.

Canonical statement:
StreamingOS shall be designed
so that multilingual operation
is possible through shared language-aware structures.

# ============================================================
# 4. MULTI-CURRENCY SUPPORT
# ============================================================

StreamingOS shall support multi-currency handling.

In StreamingOS,
currency handling shall distinguish:
- internal canonical monetary meaning
- display currency meaning
- beneficiary-side currency meaning
- nation-currency interpretation inside Civilization

For Civilization-side use,
national currency display shall be supported
according to the nation context
of the relevant beneficiary or display subject.

Multi-currency support applies to:
- tips
- gifts
- settlement summaries
- creator revenue summaries
- corporate revenue visibility
- monetization-related notifications where relevant

Canonical statement:
StreamingOS shall be designed
to support multi-currency interpretation
and nation-aware currency display.

# ============================================================
# 5. PLATFORM / INTERNATIONAL SUPPORT INTERPRETATION
# ============================================================

StreamingOS is not only
a stream publication platform.

It is also:
- a creator platform
- a viewer platform
- a moderation platform
- a notification platform
- a corporate oversight platform

Therefore,
cross-device support,
multilingual support,
and multi-currency support
must apply across these major platform surfaces,
not only to limited viewer-side pages.

# ============================================================
# 6. FINAL ADDITIVE STATEMENT
# ============================================================

StreamingOS shall be treated
as a multilingual,
multi-currency,
cross-device platform
supporting:
- iPhone
- Android
- PC
- tablet

This interpretation is canonical
for future design refinement.
