# ============================================================
# 1. PURPOSE
# ============================================================

This document gives the model overview
for StreamingOS.

The model layer defines
the canonical data meaning
for creator,
viewer,
moderation,
notification,
corporate oversight,
publication,
reaction,
and monetization structures.

# ============================================================
# 2. MODEL POSITION
# ============================================================

StreamingOS model must support:
- creator-side canonical structures
- viewer-side continuity structures
- moderation and restoration structures
- notification and alert structures
- company / affiliation oversight structures
- tip / monetization structures
- archive / clip / publication structures

# ============================================================
# 3. CROSS-DEVICE MODEL SUPPORT
# ============================================================

StreamingOS model shall be designed
for cross-device usage.

Supported device classes:
- iPhone
- Android
- PC
- tablet

Model implication:
- canonical data meaning must not depend
  on a single device class
- creator,
  viewer,
  moderation,
  notification,
  and corporate oversight structures
  must remain device-neutral
- device differences belong to interface/runtime concerns,
  not to canonical data meaning

# ============================================================
# 4. INTERNATIONAL MODEL SUPPORT
# ============================================================

StreamingOS model shall support:
- multilingual operation
- multi-currency handling
- nation-aware currency display inside Civilization

Model implication:
- language-aware display support must be model-compatible
- notification and review wording references
  must be language-aware compatible
- monetization and revenue structures
  must remain compatible with display currency handling
- canonical monetary meaning must remain distinct
  from localized or display-oriented currency meaning

# ============================================================
# 5. MODEL DISTINCTIONS
# ============================================================

StreamingOS model must preserve:
- principal vs performer
- ownership vs participation
- archive existence vs external publication
- formal tipping vs Ai(human) auto tipping
- creator self-state vs public channel state
- viewer continuity vs recommendation/ranking output
- moderation restriction vs restoration
- official company ownership vs affiliated participation
- canonical monetary meaning vs display currency meaning
