# ============================================================
# 1. INTEGRATED PURPOSE
# ============================================================

This document integrates
the interface layer of StreamingOS.

It defines how StreamingOS surfaces
are presented across creator,
viewer,
moderation,
notification,
and corporate oversight use cases.

# ============================================================
# 2. INTEGRATED INTERFACE INTENT
# ============================================================

StreamingOS interface must never be reduced to:
- viewer watch screen only
- creator upload screen only
- inbox only
- company dashboard only

StreamingOS interface must instead be understood as:
- creator platform interface
- viewer platform interface
- moderation interface
- notification interface
- corporate oversight interface
- monetization visibility interface

# ============================================================
# 3. CURRENT INTERFACE SCOPE
# ============================================================

StreamingOS interface currently includes:

creator-side:
- creator my page
- channel management
- upload / studio
- creator analytics
- creator settings

viewer-side:
- home / discovery
- search
- waiting room
- live watch
- archive watch
- clip watch
- follow / history / playlist / queue

moderation-side:
- AI review
- human review
- report handling
- comment moderation
- appeal
- restoration

notification-side:
- creator notifications
- viewer notifications
- review alerts
- escalation alerts

corporate-side:
- official company channels
- affiliated streamer visibility
- oversight dashboard
- issue visibility
- monetization visibility

# ============================================================
# 4. CROSS-DEVICE INTERFACE INTERPRETATION
# ============================================================

StreamingOS is a cross-device interface platform.

Supported device classes:
- iPhone
- Android
- PC
- tablet

Interface implication:
- creator and viewer surfaces must remain usable
  across smartphone,
  PC,
  and tablet
- moderation,
  analytics,
  review,
  and oversight surfaces must have strong support
  on PC and tablet
- interface design must not assume one-device-only usage

# ============================================================
# 5. INTERNATIONAL INTERFACE INTERPRETATION
# ============================================================

StreamingOS interface is:
- multilingual
- multi-currency
- nation-aware in currency display

This means interface design must include shared support for:
- language-aware labels
- language-aware notifications
- language-aware moderation/review wording
- currency-aware monetization summaries
- nation-aware display conversion
- separation of canonical money meaning and display currency meaning

# ============================================================
# 6. INTERFACE DISTINCTIONS
# ============================================================

StreamingOS interface must preserve:
- creator self-interface vs public channel interface
- viewer continuity interface vs discovery interface
- moderation workspace vs public-facing outcome
- notification inbox vs review/escalation alerts
- official company ownership interface vs affiliated participation interface
- canonical monetary meaning vs display currency meaning
