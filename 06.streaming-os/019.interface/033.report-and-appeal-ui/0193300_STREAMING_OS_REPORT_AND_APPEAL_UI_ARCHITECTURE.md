# ============================================================
# 1. PURPOSE
# ============================================================

This document defines the UI architecture
for reporting and appeal behavior in StreamingOS.

# ============================================================
# 2. UI GOAL
# ============================================================

The UI must support:
- report target selection
- report reason submission
- report status visibility
- appeal submission
- re-review request submission
- appeal status visibility
- outcome visibility

# ============================================================
# 3. MAIN SCREEN BLOCKS
# ============================================================

Recommended blocks:
- report submission form
- report status panel
- appeal submission form
- appeal history panel
- re-review request panel
- outcome summary panel

# ============================================================
# 4. CANONICAL FIXED STATEMENT
# ============================================================

The report and appeal UI
shall expose reporting and appeal behavior
as explicit platform actions
with explicit status visibility.
