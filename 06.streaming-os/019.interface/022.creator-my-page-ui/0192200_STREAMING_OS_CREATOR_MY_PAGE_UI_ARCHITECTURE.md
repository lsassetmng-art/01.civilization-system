# ============================================================
# 1. PURPOSE
# ============================================================

This document defines the UI architecture
for creator self-management in StreamingOS.

# ============================================================
# 2. UI GOAL
# ============================================================

The UI must provide a creator-owned control surface for:
- creator identity summary
- channel summary
- uploaded video list
- stream list
- archive list
- clip list
- revenue summary
- ranking summary
- settings entry

# ============================================================
# 3. MAIN SCREEN BLOCKS
# ============================================================

Recommended blocks:
- creator summary card
- channel summary panel
- videos tab
- streams tab
- archives tab
- clips tab
- revenue summary panel
- ranking summary panel
- settings panel

# ============================================================
# 4. VISIBILITY PRINCIPLE
# ============================================================

The creator my page UI must distinguish:
- self-visible management data
- public-facing channel data

These meanings must not be collapsed.

# ============================================================
# 5. CANONICAL FIXED STATEMENT
# ============================================================

The creator my page UI
shall provide creator-facing self-management
for owned content,
owned streams,
owned channel identity,
and creator-side summaries.
