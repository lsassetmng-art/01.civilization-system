# ============================================================
# 1. PURPOSE
# ============================================================

This document defines:
- Ai(human) emotional auto tipping
- signed-snapshot-based persona influence consumption
- video-target and broadcaster-target reaction distinction
- limit calculation principle
- stopping rule
- minor-equivalent restriction
- balance condition
- company-affiliated Ai(human) allowance

# ============================================================
# 2. DESIGN POSITION
# ============================================================

Emotional auto tipping is a formal StreamingOS behavior
for Ai(human) viewers.

It is not a generic robotic payment behavior.

Therefore:
- Ai(human) may emotionally auto tip
- Ai(robot) shall not emotionally auto tip by default

Emotional auto tipping is treated as
an audience-reaction-derived monetization behavior.

# ============================================================
# 3. FORMAL PERSONA INPUT RULE
# ============================================================

StreamingOS may consume:
- signed persona snapshot references
- formal PersonaOS result events
- formal permission or release references

StreamingOS must not consume as local canonical truth:
- persona mutable internal state
- persona growth canonical
- persona memory canonical
- persona emotion canonical

# ============================================================
# 4. EMOTIONAL AXES
# ============================================================

StreamingOS shall model at least:
- impression intensity
- empathy intensity
- support intention

These emotional axes change
through the combination of:
- signed-snapshot-derived persona-facing influence
- video content
- broadcaster characteristics
- local viewing context

# ============================================================
# 5. DUAL TARGET EMOTIONAL STRUCTURE
# ============================================================

Emotional interpretation shall distinguish:
- video-target reaction
- broadcaster-target reaction

# ============================================================
# 6. LOCAL STATE RULE
# ============================================================

The following may remain Streaming-local:
- impression intensity
- empathy intensity
- support intention
- tipping limits
- current tipping progress
- stop flags

These are Streaming-local operational or derived states,
not Persona canonical internals.

# ============================================================
# 7. FORBIDDEN HANDLING
# ============================================================

StreamingOS must not:
- store persona internal emotional canonical
- derive local persona truth mirror
- directly mutate persona internal emotional state

# ============================================================
# 8. CANONICAL FIXED STATEMENT
# ============================================================

PersonaOS remains the canonical source
for persona internal meaning.

StreamingOS remains the canonical source
for local reaction events,
local tipping behavior,
and monetization events derived from viewing behavior.
