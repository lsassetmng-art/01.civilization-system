# ============================================================
# 1. PURPOSE
# ============================================================

This document defines:
- comment moderation meaning
- report meaning
- appeal meaning
- re-review meaning
- restoration meaning

# ============================================================
# 2. DESIGN POSITION
# ============================================================

StreamingOS governance must extend beyond session/asset restriction
into interaction-side moderation and restoration.

# ============================================================
# 3. COMMENT MODERATION PRINCIPLE
# ============================================================

Comment moderation should preserve explicit states for:
- visible
- hidden
- removed
- restricted
- restored

These should not be collapsed into silent deletion.

# ============================================================
# 4. REPORT PRINCIPLE
# ============================================================

Reports should preserve:
- report target
- report reason
- report status
- review path
- resolution outcome

# ============================================================
# 5. APPEAL / RE-REVIEW PRINCIPLE
# ============================================================

Appeals and re-review should preserve:
- explicit request existence
- request status
- new decision outcome
- history preservation

# ============================================================
# 6. RESTORATION PRINCIPLE
# ============================================================

Restoration should remain explicit and review-backed.

A restored state must not erase prior restriction history.

# ============================================================
# 7. CANONICAL FIXED STATEMENT
# ============================================================

StreamingOS shall provide native governance structures
for comment moderation,
reports,
appeals,
re-review,
and restoration,
while preserving explicit restriction and history meanings.
