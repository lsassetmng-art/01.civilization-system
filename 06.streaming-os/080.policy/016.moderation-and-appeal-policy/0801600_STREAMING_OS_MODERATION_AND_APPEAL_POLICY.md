# ============================================================
# 1. POLICY PURPOSE
# ============================================================

This policy defines moderation,
report,
appeal,
re-review,
and restoration rules inside StreamingOS.

# ============================================================
# 2. COMMENT MODERATION POLICY
# ============================================================

Comment moderation must preserve explicit states such as:
- visible
- hidden
- removed
- restricted
- restored

Moderation should not default to silent disappearance.

# ============================================================
# 3. REPORT POLICY
# ============================================================

StreamingOS shall support explicit report submission.

Report existence,
review status,
and resolution outcome
must remain explicit.

# ============================================================
# 4. APPEAL / RE-REVIEW POLICY
# ============================================================

StreamingOS shall support explicit appeal
and re-review request handling
for relevant moderation or review outcomes.

Appeal existence and final outcome must remain visible.

# ============================================================
# 5. RESTORATION POLICY
# ============================================================

Restoration must remain explicit and review-backed.

Restored state must not erase prior restriction history.

# ============================================================
# 6. CANONICAL FIXED STATEMENT
# ============================================================

StreamingOS shall provide explicit moderation,
report,
appeal,
re-review,
and restoration handling
as native platform policy behavior.
