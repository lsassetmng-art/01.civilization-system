# ============================================================
# 1. OPERATIONS PURPOSE
# ============================================================

This operations domain defines
day-to-day moderation and appeal handling
inside StreamingOS.

# ============================================================
# 2. PRIMARY RESPONSIBILITIES
# ============================================================

Moderation and appeal operations is responsible for:
- supervising report intake
- supervising comment moderation queues
- supervising appeal and re-review queues
- supervising restoration follow-up
- preserving history visibility for restriction and restoration

# ============================================================
# 3. RESTORATION HANDLING
# ============================================================

Operations should ensure:
- restoration requests remain visible
- restoration decisions are explicit
- restored states do not erase prior restriction history
