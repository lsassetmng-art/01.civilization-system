# APPLICATION COMMON COMPONENT ORIGIN EVIDENCE INTAKE POLICY CANONICAL

status: canonical
system: CommonOS
layer: development

## Purpose
This document defines how CommonOS should intake and evaluate origin-side evidence for application common component candidates.

## Accepted origin-side inputs
- extraction ledgers
- origin-side audits
- source evidence notes
- promotion review notes
- naming conflict reports
- current-wave and hold recommendations

## Intake steps
1. register origin evidence
2. map candidate to a provisional CommonOS family name
3. check for same-family duplication
4. confirm layer classification
5. evaluate second-consumer possibility
6. route to adoption review

## Fixed origin interpretation
StaticArtOS is currently an important origin evidence source, but it is not the active canonical home of application common component design.

## Intake rule
Origin evidence is authoritative for provenance, not for CommonOS canonical ownership.
