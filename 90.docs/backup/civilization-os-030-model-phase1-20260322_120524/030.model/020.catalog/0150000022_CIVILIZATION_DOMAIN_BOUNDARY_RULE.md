# ============================================================
# PURPOSE
# ============================================================

Define ownership and separation rules
between civilization domains.


# ============================================================
# PRINCIPLE
# ============================================================

one concept
=
one owning domain

cross-domain meaning
=
relation
not duplicate ownership


# ============================================================
# RULES
# ============================================================

a model belongs to exactly one domain

cross-domain interactions must be relation-based

country-specific variants are data, not domains

policy variants are policy content, not new domains

market-specific implementations do not justify
duplicate concept ownership


# ============================================================
# EXAMPLES
# ============================================================

tax_system
belongs to taxation

currency
belongs to currency

payment_network
belongs to payment

company
belongs to company

cross-border company behavior
does not move company into trade domain


# ============================================================
# VIOLATIONS
# ============================================================

same concept under multiple domains
hidden ownership transfer
duplicate synonyms across domains
country-specific fragmentation as model inflation
