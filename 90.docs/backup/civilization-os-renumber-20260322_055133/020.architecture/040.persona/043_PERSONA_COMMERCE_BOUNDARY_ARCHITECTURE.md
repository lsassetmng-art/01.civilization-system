# PERSONA COMMERCE BOUNDARY ARCHITECTURE

status: canonical
layer: architecture
domain: persona
owner: Boss
prepared_by: Zero

## PURPOSE
Define the commerce boundary for Persona-derived products.

## PERSONA OS OWNS
- persona original data
- snapshot original data
- release eligibility
- license authority
- access authority

## CIVILIZATION OS OWNS
- product catalog entry
- purchase flow
- transaction flow
- payment handling
- reward distribution

## RULE
A Persona-derived item is sold in CivilizationOS
but remains governed by PersonaOS rights and access state.
