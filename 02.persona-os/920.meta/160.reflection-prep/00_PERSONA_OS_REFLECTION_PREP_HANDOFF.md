# 2. MOST IMPORTANT SHARED RULES
# ============================================================

The following remain owned by PersonaOS:

- identity authority
- current state authority
- governed growth truth
- snapshot authority
- security gate truth
- governance decision truth
- canonical visual truth
- committed mutation truth

Downstream systems may hold:
- references
- projections
- derived views
- runtime/session state
- transport traces
- support/debug records
- render outputs
- business/world/context truth of their own domains

But downstream systems must not silently promote those
into PersonaOS authority truth.

VisualRuntime:

- may resolve/render/cache/display visual inputs
- must not convert render artifacts into Persona truth

# ============================================================
# 4. SHARED STAGE-SEPARATION RULE
# ============================================================

All downstream targets must preserve:

1. request accepted
2. security verified
3. governance approved
4. canonical mutation committed
5. response returned
6. delivery observed or render executed
7. support logged

Strict separation:
- accepted is not verified
- verified is not approved
- approved is not committed
- committed is not returned
- returned is not delivered
- render executed is not authoritative truth
- delivered is not authoritative truth
- support logged is not authoritative truth

# ============================================================
# 5. MOST IMPORTANT MISREFLECTION RISKS
# ============================================================

Do not allow:
- local label -> Persona identity authority
- local workflow/activity/UI state -> Persona state authority
- local observations -> Persona growth truth
- copied/exported/cached data -> Persona snapshot authority
- local validation -> Persona security truth
- local approval -> Persona governance truth
- response receipt -> canonical completion
- delivery observation -> committed mutation truth
- render output -> canonical visual truth
- support/debug traces -> authority records
