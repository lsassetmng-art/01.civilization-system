# ============================================================
# 2. MOST IMPORTANT SHARED RULES
# ============================================================

The following remain owned by PersonaOS:

- identity authority
- current state authority
- governed growth truth
- snapshot authority
- security gate truth
- governance decision truth
- canonical visual truth
- committed mutation truth

Foundation layers may hold:
- canonical input references
- resolved execution plans
- runtime/session state
- buffers/caches
- rendered/output artifacts
- diagnostics/performance/support traces

But foundation layers must not silently promote those
into PersonaOS authority truth.

# ============================================================
# 3. TARGET ROLE SUMMARY
# ============================================================

VisualRuntime:
- PersonaOS foundation-side visual execution layer
- may resolve/render/cache/display/support visual execution artifacts
- must not convert execution artifacts into Persona truth

# ============================================================
# 4. SHARED STAGE-SEPARATION RULE
# ============================================================

All foundation targets must preserve:

1. input accepted
2. security verified where applicable
3. governance approved where applicable
4. canonical mutation committed where relevant
5. visual input resolved
6. render executed
7. output displayed or emitted
8. diagnostics/support logged

Strict separation:
- accepted is not committed
- committed is not input resolved
- input resolved is not authoritative truth
- render executed is not authoritative truth
- output displayed is not authoritative truth
- diagnostics/support logged is not authoritative truth

# ============================================================
# 5. MOST IMPORTANT MISREFLECTION RISKS
# ============================================================

Do not allow:
- render output -> canonical visual truth
- execution identifiers -> Persona identity authority
- runtime expression state -> Persona state authority
- appearance deltas -> Persona growth truth
- cached/exported render artifacts -> Persona snapshot authority
- local validation -> Persona security truth
- local display eligibility -> Persona governance truth
- diagnostics/support traces -> authority records
