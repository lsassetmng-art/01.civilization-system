# 1. WHAT THIS REFLECTION SET DOES
# ============================================================

This reflection-prep set tells targets:

- what they may consume from PersonaOS
- what they may expose or cache locally
- what they may operationally support
- what they may execute at foundation/runtime level
- what they must never silently treat as authority truth
- how stage separation must remain visible in runtime,
  workflow, transport, rendering, and support layers

This set is not:
- a replacement for PersonaOS canonical domains
- a permission to duplicate Persona authority outside PersonaOS
- a justification to collapse support/runtime/render data
  into canonical truth

# ============================================================
# 2. SHARED PERSONAOS AUTHORITY RULES
# ============================================================

The following remain owned by PersonaOS:

- identity authority
- current state authority
- governed growth truth
- snapshot authority
- security gate truth
- governance decision truth
- committed mutation truth
- canonical visual truth

Targets may hold:
- references
- projections
- derived views
- local caches
- runtime/session state
- transport traces
- support/debug records
- render outputs
- business/world context records
- foundation-side execution artifacts

But targets must not silently promote those
into PersonaOS authority truth.

# ============================================================
# 4. SHARED STAGE-SEPARATION RULE
# ============================================================

All reflection targets must preserve:

1. request accepted
2. security verified
3. governance approved
4. canonical mutation committed
5. response returned
6. delivery observed or render executed
7. support logged

Strict separation:
- accepted is not verified
- verified is not approved
- approved is not committed
- committed is not returned
- returned is not delivered
- render executed is not authoritative truth
- delivered is not authoritative truth
- support logged is not authoritative truth

This rule must remain visible in:
- UI
- local models
- transport traces
- workflow state
- render state
- diagnostics/support logs

# ============================================================
# 5. SHARED MISREFLECTION RISKS
# ============================================================

major_shared_risks:
- local label becomes identity authority
- local workflow/activity/UI state becomes Persona state authority
- local observations become growth truth
- copied/exported/cached data becomes snapshot authority
- local validation becomes Persona security truth
- local approval becomes Persona governance truth
- response receipt becomes canonical completion
- delivery observation becomes committed truth
- render output becomes canonical visual truth
- support/debug traces become authority records

# ============================================================
# 6. SHARED IMPLEMENTATION RULES
# ============================================================

All target implementations should follow:

rule_1:
Always label local values as one of:
- canonical reference
- derived projection
- runtime/session state
- integration/transport state
- support/debug trace
- render artifact
- business/world-owned truth
- foundation execution artifact

rule_2:
Never serialize convenience state as Persona canonical truth.

rule_3:
Do not let local/offline/render/workflow assumptions silently
overwrite last confirmed PersonaOS truth.

rule_4:
When Persona-linked status is unknown, stale, or unresolved,
preserve unresolved state instead of fabricating completion.

rule_5:
Any displayed "success" must be clearly scoped:
- request accepted
- response returned
- delivery observed
- render executed
- canonical commit confirmed
