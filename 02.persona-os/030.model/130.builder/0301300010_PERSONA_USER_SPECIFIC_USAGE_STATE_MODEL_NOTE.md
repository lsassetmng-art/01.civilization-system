# PURPOSE

Define the conceptual separation between shared Persona truth
and user-specific usage variation.


# SHARED BASE PERSONA

Shared base Persona includes:

- core personality
- core values
- official profile core
- official qualifications
- official skills
- official visual
- official voice


# USER-SPECIFIC USAGE STATE

User-specific usage state may include:

- affinity
- trust
- familiarity
- emotional closeness
- shared history score
- response tuning
- learned preference adaptation


# RULE

User-specific usage state must not overwrite
shared base Persona truth.


# FINAL DEFINITION

PersonaOS must model shared Persona truth
and user-specific usage variation as separate layers.
