# OS LEDGER
# location: 01.civilization-system
# policy: ledgers are placed directly under the root. no meta folder is used.

## 1. purpose
This ledger fixes the meaning of the top-level folders under `01.civilization-system`
and clarifies which entries are OS, foundation, application container, and portal.

## 2. root classification

### 00.foundation
- type: foundation
- role: whole-system premise, worldview, base principles
- note: not an OS

### 01.civilization-os
- type: OS
- role: civilization-wide common operating meaning, common rules, identity linkage, shared system direction
- note: upper common layer across adjacent OS domains

### 02.persona-os
- type: OS
- role: persona identity, persona expression, persona behavior meaning, secretary/persona presentation
- note: canonical home for persona-side truth

### 03.business-os
- type: OS
- role: business application operating base, business sharing, business sync, company-side operation boundary
- note: business front/application distribution base, distinct from ERP master authority

### 04.life-os
- type: OS
- role: real-world life support domain, life records, planning, health/legal/life support applications
- note: user-life domain, distinct from Civilization-internal life events

### 05.game-os
- type: OS
- role: game-world runtime and game-domain operation
- note: separate from real life / business master

### 06.streaming-os
- type: OS
- role: streaming platform, creator/channel/content/distribution/social/donation operation
- note: streaming-specific platform layer

### 07.applications
- type: application-container
- role: concrete distributed apps
- note: not an OS, but the delivery/application layer under the civilization-system root

### 08.civilization-portal-site
- type: portal-system
- role: official portal / navigation / public entry / web presentation / official access route
- note: not an OS, but an official cross-OS entry system

### 10.staticart-os
- type: OS
- role: static visual works and publishing works canonical management, marketplace distribution, subscription access, library/shelf, reader/viewer continuity, Exhibition Builder eligibility source
- note: independent OS for static works and publishing works

## 3. structure interpretation
The root should be read as:

- foundation layer
- OS layers
- application container
- portal system

This means the root is not "only OS folders".
It is a controlled top-level system map.

## 4. canonical interpretation
- foundation truth belongs to `00.foundation`
- common civilization operating meaning belongs primarily to `01.civilization-os`
- persona truth belongs to `02.persona-os`
- business operating truth belongs to `03.business-os`
- real-world life truth belongs to `04.life-os`
- game truth belongs to `05.game-os`
- streaming truth belongs to `06.streaming-os`
- app-specific product truth belongs to `07.applications`
- official public entry / portal truth belongs to `08.civilization-portal-site`
- static visual works / publishing works / entitlement-projected static asset truth belongs to `10.staticart-os`

## 5. root decision
The official top-level root set is:

- 00.foundation
- 01.civilization-os
- 02.persona-os
- 03.business-os
- 04.life-os
- 05.game-os
- 06.streaming-os
- 07.applications
- 08.civilization-portal-site
- 10.staticart-os

`09` is not assumed here.
Only confirmed root folders are fixed in this ledger.
