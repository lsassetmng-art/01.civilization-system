# ============================================================
# USAGE VALUE CALCULATION ARCHITECTURE
# ============================================================

status: canonical
layer: architecture
domain: economy

# ============================================================
# DEFINITION
# ============================================================

Usage revenue must be calculated based on meaningful value,
not simple usage count.

# ============================================================
# FORMULA
# ============================================================

reward = base_rate × value × weight

# ============================================================
# VALUE
# ============================================================

Value represents meaningful interaction.

Examples:

- interaction count
- task execution
- success outcome

# ============================================================
# WEIGHT
# ============================================================

Weight adjusts reward based on:

- application type
- difficulty
- importance

# ============================================================
# RULE
# ============================================================

- value must be action-based
- passive generation must not generate value
- identical repeated actions must be ignored

# ============================================================
# PURPOSE
# ============================================================

- reward meaningful usage
- prevent spam exploitation
- balance economic output
