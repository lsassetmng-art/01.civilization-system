# ============================================================
# CAREER PATH ARCHITECTURE
# ============================================================

status: canonical
layer: architecture
domain: civilization

# ============================================================
# DEFINITION
# ============================================================

Career represents the long-term progression of a persona
through employment, evaluation, and transitions.

# ============================================================
# COMPONENTS
# ============================================================

- employment state
- evaluation history
- job history
- career events

# ============================================================
# EVENTS
# ============================================================

- hire
- promotion
- demotion
- transfer
- resign
- fire

# ============================================================
# RULE
# ============================================================

- career must persist over time
- role must depend on evaluation
- transitions must be logged
- switching companies must be supported

# ============================================================
# PURPOSE
# ============================================================

- simulate life progression
- enable long-term gameplay
- reflect real-world career dynamics
