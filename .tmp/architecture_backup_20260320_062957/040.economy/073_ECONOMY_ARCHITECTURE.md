# ============================================================
# ECONOMY ARCHITECTURE
# ============================================================

status: canonical
layer: architecture

definition:
Economy architecture defines the value circulation system
connecting persons, companies, markets, and governments.

purpose:
- model income, expenditure, ownership, taxation, and capital flow
- provide structured economic progression for Civilization

core_subdomains:
- citizenship linkage
- currency
- bank
- employment
- market
- company
- stock
- ma_tob
- exchange
- treasury
- revenue_ledger

rules:
- economy is event-driven
- ownership and value must have lineage
- law constrains economy, but does not erase economic traceability
- interface must not directly mutate economic state

constitution_alignment:
- explicit data lineage
- no silent balance mutation
- lawful boundary separation required

final_rule:
Civilization economy must remain explicit, auditable, and structurally separated
from UI shortcuts and hidden conversions.
