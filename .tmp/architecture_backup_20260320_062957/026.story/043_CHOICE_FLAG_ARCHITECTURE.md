# ============================================================
# CHOICE FLAG ARCHITECTURE
# ============================================================

status: canonical
layer: architecture
domain: story

# ============================================================
# DEFINITION
# ============================================================

Choice flags represent the internal state of the story.

They are modified by user choices.

# ============================================================
# TYPES
# ============================================================

- boolean
- number
- enum

# ============================================================
# STORAGE
# ============================================================

Flags must be stored in save data.

# ============================================================
# SETTING FLAGS
# ============================================================

Choices may set flags.

# ============================================================
# CONDITIONS
# ============================================================

Scene transitions may depend on flags.

# ============================================================
# RULES
# ============================================================

- flags must not affect world data
- flags exist only within story
- flags must be deterministic

# ============================================================
# BRANCH CONTROL
# ============================================================

- branching allowed
- final convergence required

# ============================================================
# FINAL RULE
# ============================================================

Flags define the player's path
without altering the world.
