Defines runtime behavior for design document generation.

runtime_steps:
1. resolve design intent
2. resolve target design language
3. resolve applicable rules
4. inspect reusable component catalog if needed
5. generate design structure
6. generate design body
7. classify warnings / gaps
8. package as proposal

generation_modes:
- free input expansion
- guided design completion
- standard template generation
- full-generation format transformation
- imported design normalization

outputs:
- design proposal
- design warnings
- gap list
