# ============================================================
# 1. DEFINITION
# ============================================================

App Development Studio is an independent, configurable,
business-oriented development execution application.

It supports design authoring, imported design parsing,
readiness-based auto/manual routing, rule-driven generation,
reusable component reuse, structured DB targeting,
review/approval/apply control, Git-linked delivery,
testing support, bug/version/progress management,
multilingual operation, optional build execution,
and guarded execution control.

# ============================================================
# 2. MAIN VALUE
# ============================================================

The application allows one target application to be designed,
generated, reviewed, applied, versioned, tested,
tracked, and delivered inside one product.

# ============================================================
# 3. DESIGN INPUT AND ROUTING
# ============================================================

Supported input modes:
- free_input
- guided_input
- standard_template
- full_generation_format
- imported_design_documents

Readiness decisions:
- full_ready
- full_ready_with_warning
- manual_required
- blocked

# ============================================================
# 4. GENERATION AND CONTROL
# ============================================================

Major generation/control surfaces:
- design docs
- Java source
- SQL
- scripts
- package structures
- Java unit tests
- higher-level test item sheets
- release outputs
- Git change sets

Control surfaces:
- review
- approval
- apply
- backup
- rollback
- Git commit/push/tag
- build

# ============================================================
# 5. RULES / REUSE / DB TARGETING
# ============================================================

The application supports:
- mutable and versionable rule profiles
- reusable component catalog
- BusinessOS shared component reuse
- user-private component reuse
- structured DB connection profiles
- AI-friendly connection selection

# ============================================================
# 6. TEST / BUG / VERSION / PROGRESS
# ============================================================

The application supports:
- Java unit test auto generation
- integration/system/acceptance test item sheets
- test progress management
- manual test result/status editing
- bug list management
- configurable artifact versioning
- one-app project progress management

# ============================================================
# 7. MULTILINGUAL / BUILD
# ============================================================

Language control includes:
- UI language
- input assist language
- artifact-specific output languages
- fallback language

Build modes include:
- disabled
- enabled
- if_available
- external_only

# ============================================================
# 8. IMPLEMENTATION-READY DETAIL DESIGN
# ============================================================

The design set additionally includes implementation-ready direction for:
- API exact contract
- DB physical schema
- UI exact item definition
- authority matrix
- readiness threshold profiles

# ============================================================
# 9. EXECUTION CONTROL DETAIL DESIGN
# ============================================================

The design set additionally includes guarded execution detail for:
- SQL safety classification
- Git exact control table
- review escalation defaults
- execution matrix MVP rows
- SQL/Git/escalation runtime-flow-policy-interface-implementation alignment

This means execution behavior is now clearer for:
- SQL execution
- protected Git push
- escalation from light control to strong control
- actor-role-based action permission

# ============================================================
# 10. MVP IMPLEMENTATION ASSETS
# ============================================================

Current MVP-ready assets include:

- MVP execution control tables

system: applications
application: App Development Studio
owner: Boss

purpose:
Defines the main user-facing interfaces of App Development Studio.

interface_goal:
To let the user control design input, generation, review, apply,
Git, testing, bug tracking, progress, and settings in one application.

main_interface_areas:
- request input
- design input mode selection
- generation readiness result
- target and rule selection
- proposal list and diff viewer
- review / approval / apply
- backup confirmation
- Git controls
- bug list and bug detail
- version settings
- language settings
- test progress
- project progress
- build settings
- test manual edit

interface_principles:
- control should be explicit
- mode selection should be visible
- warnings should be understandable
- editable items should remain editable
- internal AI assistance should not hide key decisions
