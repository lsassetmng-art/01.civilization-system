purpose:
Defines policy control principles for App Development Studio.

policy_goal:
To let the user choose operation speed and control level,
while still preserving clarity, traceability, and configurable safety.

policy_domains:
- rule governance
- approval and review
- backup and rollback
- Git protection
- environment safety
- reusable component reuse
- connection safety
- test generation and management
- version and bug handling
- multilingual behavior
- build execution
- manual overrides

policy_principles:
- configurability is first-class
- unsafe actions may be escalated
- user intent is respected within policy boundaries
- review and approval are not identical
- build is optional
- multilingual operation is normal
- progress and bug visibility are required
