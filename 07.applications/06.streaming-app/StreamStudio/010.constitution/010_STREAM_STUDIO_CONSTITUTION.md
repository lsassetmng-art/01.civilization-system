# 1. MISSION

StreamStudio is the official creator production and
publishing operations application for StreamingOS.

It exists to let creators and production teams manage:
- upload
- metadata
- thumbnail
- subtitle
- review
- approval
- collaboration
- publication
- external push extension
- revenue split setup

# 2. CORE PRINCIPLES

1. StreamingOS is the source of truth.
2. StreamStudio is the creator operating surface.
3. Draft editing and canonical publication are separate.
4. Review is separate from approval.
5. Approval is separate from publication.
6. External push is downstream only.
7. External push success never replaces StreamingOS truth.
8. Rights and split basis must be auditable.
9. Mobile is light-operation-first.
10. Desktop and tablet are heavy-operation-first.
11. Multilingual support is first-class.
12. Multicurrency support is first-class.

# 3. TRUTH BOUNDARY

Canonical truth belongs to StreamingOS for:
- asset ownership
- publication truth
- approval trail truth
- monetization linkage truth
- rights and lifecycle truth

StreamStudio may hold:
- creator drafts
- creator UI state
- upload working state
- review work state
- split preparation state

# 4. EXTERNAL PLATFORM BOUNDARY

External platforms such as YouTube are treated as:
- publication destinations
- external connector targets
- non-canonical mirrors

They are never treated as:
- source of truth
- ownership authority
- canonical publication authority for StreamingOS-native assets

# 5. GOVERNANCE PRINCIPLES

- Every critical action must remain reconstructible.
- Review, approval, publish, and split changes must be traceable.
- Policy override must remain explicit.
- Failure and retry states must remain visible.
