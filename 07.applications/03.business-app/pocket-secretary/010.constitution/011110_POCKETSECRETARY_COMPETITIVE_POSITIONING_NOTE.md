purpose:
Clarifies the differentiator that should be made explicit in the
current PocketSecretary design without rewriting its existing core.

positioning_summary:
- PocketSecretary is not only a persona-hosted assistant surface.
- Its durable value is the conversion of conversation into
  structured follow-through across schedule, task, reminders,
  library, transit, expense, and business-card related actions.
- The differentiator is not persona display alone, but assistant-led
  continuity from message, to plan, to reminder, to completion.

preserved_core:
- persona-centric UI rendering
- local-first stance
- assistant application identity
- existing schedule / task / reminder / library role
- existing optional online extension direction
- existing Civilization and OS relation

differentiator_to_make_explicit:
- conversation-to-action orchestration
- assistant memory for unfinished commitments
- daily briefing and follow-through visibility
- unified action timeline across assistant-managed domains

competitive_contrast:
- not merely a chat front-end
- not merely a calendar shell
- not merely a task list wrapper
- positioned as an assistant continuity layer that remembers,
  proposes, schedules, reminds, and follows through

- no payload replacement
