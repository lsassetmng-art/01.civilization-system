status: canonical-draft
layer: model
application: App Development Studio

purpose:
Defines a retainable proposal unit generated by the application.

definition:
proposal is a generated candidate output or change set
that may be reviewed, retained, approved, rejected, applied, or rolled back.

minimum_fields:
- proposal_id
- request_id
- proposal_type
- proposal_title
- proposal_summary
- proposal_status
- risk_level
- diff_available
- artifact_count
- generated_at
- generated_by_mode
- created_at
- updated_at

proposal_type_examples:
- design
- source_code
- sql
- test
- package
- git_change
- mixed

proposal_status_examples:
- drafted
- reviewed
- retained
- approved
- rejected
- applied
- rolled_back

risk_level_examples:
- low
- medium
- high
- critical

relations:
- links to proposal_review
- links to proposal_retention
- links to diff_record
- links to apply_request
- links to backup_record

notes:
Proposal is the core reviewable object in the system.
