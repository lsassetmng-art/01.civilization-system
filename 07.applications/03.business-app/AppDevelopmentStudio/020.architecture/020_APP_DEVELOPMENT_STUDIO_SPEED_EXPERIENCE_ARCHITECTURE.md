# ============================================================

application: App Development Studio

prepared_by: Zero

purpose:
Adds speed-oriented experience features
while preserving the existing governance-first design.

speed_experience_position:
App Development Studio shall remain governance-strong,
but shall also improve perceived speed,
initial usability,
and “already moving” product experience.

added_features:
- Quick Start / Instant Mode
- Live Preview / Draft Preview
- One-Click Starter Pack
- Interactive Diff / Revert UX Strengthening
- Fast Mode / Governed Mode Split
- Device Role Split (smartphone vs PC)

feature_meanings:

quick_start_instant_mode:
A minimal-start route that quickly generates
an initial working draft package from a short request.

live_preview_draft_preview:
A preview route that immediately shows
generated structure, expected outputs, and draft artifacts
without waiting for full controlled execution.

one_click_starter_pack:
A fast-start package generator for MVP-level kickoff output.

interactive_diff_revert_ux_strengthening:
A clearer difference and revert experience
so controlled change feels safe rather than heavy.

fast_mode_governed_mode_split:
A two-lane model where fast draft generation
and governed adoption are explicitly separated.

device_role_split:
Smartphone is primarily for review, light edits, approval, and monitoring.
PC is primarily for generation, detailed review, Git/SQL handling, and build-related work.
