# ============================================================
# MBO RIVAL COMPARISON ARCHITECTURE
# ============================================================

status: canonical
application: Mbo
layer: architecture
owner: Boss
prepared_by: Zero

purpose:
Defines rival comparison results for Mbo
and fixes the competitive design direction.

comparison_scope:
- Kaonavi
- SmartHR
- HRBrain

comparison_positioning:
Mbo should not be designed as a simple goal sheet tool.
Mbo should compete as an execution management front
with evaluation handling and ERP linkage readiness.

# ============================================================
# 1. RIVAL OBSERVATIONS
# ============================================================

rival_observations:

  Kaonavi:
    strengths:
      - strong evaluation operation support
      - template-based evaluation handling
      - progress visibility
      - reminder support
      - interview record support
      - evaluation adjustment support
    interpretation:
      Kaonavi is strong in evaluation workflow efficiency
      and management-side visibility.

  SmartHR:
    strengths:
      - customizable evaluation sheets
      - customizable evaluation flows
      - MBO support
      - evaluator auto assignment
      - 360 feedback support
      - smartphone support
      - evaluation aggregation support
    interpretation:
      SmartHR is strong in standardized HR evaluation operation
      and practical deployment efficiency.

  HRBrain:
    strengths:
      - OKR and MBO templates
      - sheet customization
      - workflow configuration
      - progress dashboard
      - 1on1 and interview records
      - bulk reminder sending
      - smartphone support
      - aggregation and export
    interpretation:
      HRBrain is strong in easy operation,
      progress visibility, and review/meeting continuity.

# ============================================================
# 2. WHERE RIVALS ARE STRONG
# ============================================================

rival_common_strengths:
- evaluation template handling
- evaluation sheet flexibility
- workflow and approval support
- progress visibility
- interview or review records
- reminder handling
- smartphone operation
- aggregation and output support

design_meaning:
If Mbo lacks the above baseline functions,
it will be weaker than common market expectations.

# ============================================================
# 3. WHERE MBO SHOULD DIFFERENTIATE
# ============================================================

Mbo_differentiation_axes:
- objective to action-plan execution continuity
- blocker recording and next-action control
- app-to-app result linkage
- ERP transfer readiness after final confirmation
- ERP transmission history and resend control
- evaluation final-lock governance
- execution-first operation, not evaluation-only operation

design_intent:
Mbo must be stronger in
"goal -> action -> progress -> review -> finalization -> ERP transfer"
than rivals.

# ============================================================
# 4. MUST-HAVE FOLLOW FEATURES
# ============================================================

must_have_follow_features:
- MBO template support
- flexible evaluation sheets
- review workflow support
- progress dashboards
- reminder handling
- review and interview logs
- smartphone support
- aggregation visibility
- exportable records

# ============================================================
# 5. MUST-WIN DIFFERENTIATION FEATURES
# ============================================================

must_win_features:
- action-plan breakdown per objective
- blocker log and next-action log
- linkage to task and project results
- evaluation confirmation lock
- ERP outbound payload preparation
- ERP transmission history
- ERP resend management
- execution-centered dashboard
- stale progress alerting
- delayed review alerting

# ============================================================
# 6. PRODUCT STRATEGY CONCLUSION
# ============================================================

strategy_conclusion:
Mbo should not fight only as an HR evaluation cloud.
Mbo should be positioned as a
goal execution and evaluation front application
with ERP linkage capability.

product_message_direction:
- not only set goals
- not only collect evaluation sheets
- operate achievement continuously
- connect confirmed outcomes to business systems

# ============================================================
# 7. NEXT DESIGN IMPACT
# ============================================================

next_design_impacts:
- strengthen action plan model
- strengthen progress and blocker model
- strengthen review cadence design
- fix ERP exact payload later
- strengthen dashboard and alert design
- keep smartphone, pc, tablet, iphone, android support mandatory
- keep multilingual and multi-currency mandatory
